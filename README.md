## 🚀 Overview

AI coding tools (like Claude, Cursor, Lovable, v0, and Bolt) are incredible for shipping fast, but they often leave behind specific artifacts, duplicate code, and prototyping leftovers. 

**Frontend Vibe Cleanup** provides the definitive rulebooks and reference materials to clean up AI-generated noise, enforce production standards, and maintain a pristine codebase.

## 🗂️ What's Inside?

### 🧠 AI Skills & Instructions
- **[`SKILL.md`](SKILL.md)**: Core instructions and skill definitions for AI agents (Claude, Cursor, Windsurf, etc.) to follow when navigating your codebase.

### 📚 Reference Playbooks (`/references`)
- **[`vibe-tools.md`](references/vibe-tools.md)**: The exhaustive cheat sheet for identifying and acting on AI tool-specific artifacts (e.g., what to do with `.lovable`, `CLAUDE.md`, `v0-*.tsx`, etc.).
- **[`nextjs.md`](references/nextjs.md)**: Best practices, server/client component boundaries, and optimizations for modern Next.js 14+ apps (App Router).
- **[`nuxt.md`](references/nuxt.md)**: Vue/Nuxt-specific architectural guidelines, state management, and conventions.
- **[`seo.md`](references/seo.md)**: Go-to-market and metadata standards ensuring that rapidly prototyped sites are actually discoverable.

## ⚙️ How to Use

1. **Clone or Copy:** Bring these reference documents into your workspace.
2. **Feed your Agent:** Point your AI coding assistant (like Claude Code, Cursor, or Copilot) to these markdown files explicitly in your system prompt or custom instructions (e.g., `.cursorrules`).
3. **Run the Cleanup:** Ask your agent to *"Run a vibe cleanup based on references/vibe-tools.md"* and watch it strip out the unused scaffolding, fix naming conventions, and prepare your app for production.

Found a new AI tool leaving weird artifacts? Discover a better Next.js pattern? Create a pull request to add it to our references!
