# Vibe Tool Artifact Reference

Complete file-by-file inventory for every major AI coding tool.
For each artifact: what it is, whether to DELETE / GITIGNORE / RENAME, and why.

---

## 🩷 Lovable (lovable.dev)

| Artifact | Action | Why |
|---|---|---|
| `.lovable` | **DELETE** | Tool config, not needed in production |
| `lovable.config.ts` / `.js` | **DELETE** | Tool config |
| `public/lovable-uploads/` | **DELETE or MIGRATE** | Replace with a real asset pipeline (Cloudinary, S3, etc.) |
| `src/integrations/supabase/` | **REVIEW** | Keep if customised; delete if untouched auto-scaffold |
| `bun.lockb` | **KEEP** | Fine if project uses Bun — only flag if team uses npm/yarn |
| Lovable logo in `public/favicon.ico` or `public/logo.png` | **REPLACE** | Swap with real brand assets |
| `// This file was auto-generated by Lovable` headers | **REMOVE** | Tool noise |
| README badge `[![Lovable](...)]` | **REMOVE** | Brand noise in public repo |
| `index.html` with `<title>Lovable App</title>` | **RENAME** | Replace with real app title |

## 🌐 General AI Tool Leftovers (Any Tool)

### Code Comments

| Pattern | Action |
|---|---|
| `// Generated by [tool]` | Remove |
| `// TODO: replace with real data` in production files | Fix the TODO or delete the file |
| `// eslint-disable` added to suppress AI-generated lint errors | Fix the actual error instead |
| `// @ts-ignore` or `// @ts-expect-error` without explanation | Fix type error, remove suppression |

### Debug Artifacts

| Pattern | Action |
|---|---|
| `console.log("test")` | Remove |
| `console.log(data)`, `console.log(response)` | Remove or replace with proper logger |
| `debugger;` statements | Remove |
| `alert(...)` for debugging | Remove |

### Placeholder / Mock Data

| Pattern | Action |
|---|---|
| `"placeholder@example.com"` in UI | Replace with env var or real copy |
| `"John Doe"`, `"Jane Smith"` | Replace with real content or typed `User` type |
| `"Lorem ipsum dolor sit amet..."` | Replace with real copy |
| `"Your Company Name"` | Replace with `process.env.NEXT_PUBLIC_APP_NAME` |
| Hardcoded demo arrays in component files | Extract to `/data/` folder or API |
| Hardcoded API keys / tokens anywhere | Move to `.env.local`, add to `.gitignore` |

### Unused Scaffolding

| Pattern | Action |
|---|---|
| `/demo`, `/test`, `/playground`, `/sandbox` routes | Delete if not gated behind auth |
| Pages that only render `<h1>Coming Soon</h1>` | Delete |
| Empty component files (`export default function() { return null }`) | Delete |
| `_temp/`, `temp/`, `scratch/` directories | Delete |
| `public/placeholder.svg`, `public/placeholder.jpg`, `public/placeholder.png` | Replace or delete |

### Bad Component Names

| Bad | Good |
|---|---|
| `Component1.tsx` | `ProductCard.tsx` |
| `Component2.tsx` | `TestimonialSlider.tsx` |
| `TestButton.tsx` | `SubmitButton.tsx` |
| `MyComponent.tsx` | `HeroSection.tsx` |
| `MyForm.tsx` | `ContactForm.tsx` |
| `NewPage.tsx` | `AboutPage.tsx` |
| `V0Component.tsx` | `PricingTable.tsx` |
| `Generated.tsx` | (describe what it renders) |

### Duplicate / Collision Files

- Check for `Button.tsx` and `button.tsx` in same directory — keep one, delete the other.
- Check for `utils.ts` and `Utils.ts` — merge and standardise.
- Check for `index.ts` + `index.tsx` in same directory — remove the one that's not used.
